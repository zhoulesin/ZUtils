# ZUtils v2 — AI 手机助手（PRD）

## 定位转变

**v1 (当前)**：AI + 动态 DEX 加载的技术框架，偏开发者工具。

**v2 (目标)**：面向普通用户的**手机智能助手**。用户用自然语言下达指令，App 理解意图并执行。功能不够？开发者或社区编写 DEX 插件扩展，用户一键安装。

## 架构变化

### v1 → v2 的演进

```
v1 架构:
  用户输入 → LLM 解析 → Workflow → 函数执行 → 结果显示

v2 架构:
  用户输入 (文本/语音/TTS反馈)
       ↓
  Intent Parser (LLM / 关键词)
       ↓
  Workflow (顺序/管道/条件/循环/定时)
       ↓
  Action Executor
       ├─ 内置 Action (闹钟/剪贴板/音量/亮度/Toast)
       ├─ 已安装插件 (天气/翻译/快递/地图)
       └─ 未安装 → 跳转插件市场 → 下载 DEX → 继续
       ↓
  结果处理 → 文本展示 / TTS / 系统通知 / 回调
```

### 新增组件

| 组件 | 职责 |
|------|------|
| **Plugin Market** | 插件发现、下载、管理、更新 |
| **Scheduler** | 定时/条件触发任务（闹钟、位置到达等） |
| **Notification Center** | 长时间任务完成通知、定时任务提醒 |
| **TTS Engine** | 语音播报结果（Android TTS） |
| **Voice Input** | 语音识别（SpeechRecognizer / ML Kit） |
| **Plugin Sandbox** | DEX 插件的安全隔离和权限管理 |
| **Action Templates** | 常用动作的快捷配置（取代手写 Workflow） |

## 插件市场

### 定位：让 App 的能力无限扩展

开发者编写 DEX 插件 → 上传到插件市场 → 用户在 App 内浏览、安装、使用。

### 插件开发流程

```
1. 开发者 fork 插件模板仓库
2. 实现 ZFunction 接口 + 声明入参/出参
3. 编译 → d8 生成 DEX
4. 上传到插件市场（携带 manifest + 版本 + 签名）
```

## 与 v1 的关系

v1 的**所有能力在 v2 中都保留**：

| v1 能力 | v2 中如何体现 |
|---------|-------------|
| 15 个内置函数 | 作为内置 Action，不可卸载 |
| DEX 动态加载 | 安装插件 = 下载 DEX + 注册到 Engine |
| 工作流引擎 | 定时任务和自动化流程使用同样的 WorkflowEngine |
| LLM 集成 | 入口统一由 LLM 或语音触发 |
| 工作流构建器 | 高级用户可自建自动化流程 |
| 权限管理 | 插件安装时声明权限，用户确认 |

## 技术栈补充

| 新增能力 | 技术方案 |
|---------|---------|
| 语音输入 | `SpeechRecognizer` / ML Kit Speech API |
| TTS 播报 | `TextToSpeech` / 系统 TTS Engine |
| 定时任务 | `WorkManager` + `AlarmManager` |
| 位置触发 | `GeofencingClient` |
| 插件市场 API | 后端接口（一期可本地 JSON 模拟） |
| 插件签名 | APK Signature Scheme / GPG 签名 |

## 非功能需求

- 首次打开到可用 < 2s
- 语音识别 < 1s（本地模型）/< 3s（云端）
- LLM 响应 < 5s
- 插件安装 < 3s（含 DEX 加载）
- 定时任务偏差 < 1min
- 离线可用：内置 Action 无网可用，插件按需下载
