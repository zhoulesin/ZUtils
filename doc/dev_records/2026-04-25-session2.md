### 3. parseQuery 增强

| 改进 | 说明 |
|------|------|
| 纯数学表达式 | `2+2`、`3*5` 等直接匹配 `^[\d+\-*/().\s]+$`，路由到 `calculate` |
| 拼写容错 | `calcute`、`cal` 等前缀经正则 `^calc[a-z]*\s*` 清理 |
| 全角符号 | `＋－×÷（）` 映射为 `+-*/()` |

### 4. 工作流参数规则

- **顺序流构建器**：第一步可选任意函数（有必填参数时留空，执行时弹出输入框）；第二步起仅显示无必填参数的函数
- **管道流构建器**：按 `isCompatible()` 筛选（输出类型 → 输入类型匹配）
- `buildWorkflow` 不再生成任何默认参数值

### 5. 执行结果区分

- `EntryType`：`TEXT`（直接输入）和 `WORKFLOW`（工作流执行）
- 历史卡片显示类型标签 `[文本]` / `[工作流]`
- 工作流结果展示入参信息
- 卡片默认收起，点击展开全部步骤详情

### 6. Toast 接受任意入参

`toast.message` 改为非必填，接受 `JsonPrimitive`、`JsonObject` 等任意类型，自动转字符串。

### 7. 权限授权流程

`WRITE_SETTINGS` 检测到未授权时，卡片底部显示红色「去授权」按钮 → 确认弹窗 → 跳转系统设置页。

### 8. LLM 集成（火山引擎）

**实现文件**：`plugin-manager/src/main/kotlin/.../llm/VolcengineLlmClient.kt`

- 使用火山引擎方舟的 `/api/coding/v3` 端点（Coding Plan 额度，不额外计费）
- API 兼容 OpenAI Chat Completions 格式，支持 Function Calling
- 自动将 16 个内置函数转为 `tools` 参数
- LLM 返回 `tool_calls` → 解析为 `Workflow` → `Engine.execute()`
- 无 `tool_calls` 时用 `toast` 显示纯文本回复
- 调用失败时回退到 `parseQuery()` 关键词匹配
- 网络请求跑在 `Dispatchers.IO`

**UI 集成**：
- TopAppBar 右侧 ⚙️ / 🔗 按钮，点击弹出 API Key 输入框
- API Key 存 `SharedPreferences`
- 输入后自动使用 LLM，未配置时仍用关键词匹配
- Logcat 过滤 `ZUtils-LLM` 可追踪执行链路
